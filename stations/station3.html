Understanding timing functions

What timing functions control

Timing functions don't change how long an animation takes ‚Äî that's duration's job. Instead, they control how the animation moves through time. Think of it like this: duration is the length of a race track, timing function is how the runner accelerates and slows down while covering that distance. Linear timing is constant speed (robotic). Ease-out accelerates fast then decelerates (responsive). Same track length, totally different running style!

The five standard curves

CSS provides five built-in timing functions. Prof. Teeters taught me to understand what each curve does before choosing one. Here's how each timing function moves:

All five timing function syntaxes:
/* Linear - constant speed throughout (robotic) */
.button-linear {
  transition: transform 300ms linear;
  /* Same speed from start to finish */
  /* Feels mechanical and unnatural */
}

/* Ease - default (balanced but not optimal) */
.button-ease {
  transition: transform 300ms ease;
  /* Starts medium, speeds up middle, slows down end */
  /* Rarely the best choice for UI */
}

/* Ease-in - slow start, fast end (feels hesitant) */
.button-ease-in {
  transition: transform 300ms ease-in;
  /* Starts slow, gradually accelerates */
  /* Good for elements leaving the screen */
}

/* Ease-out - fast start, slow end (RECOMMENDED) */
.button-ease-out {
  transition: transform 300ms ease-out;
  /* Starts fast, gradually decelerates */
  /* Best for most UI interactions */
}

/* Ease-in-out - slow both ends (good for long movements) */
.button-ease-in-out {
  transition: transform 300ms ease-in-out;
  /* Starts slow, speeds up middle, ends slow */
  /* Symmetrical curve for across-screen movement */
}

Linear: Constant speed (robotic)

Moves at exactly the same speed throughout

No acceleration or deceleration at all. Starts at full speed, maintains that speed, stops instantly. Feels mechanical and unnatural because nothing in the real world moves this way. Almost never used for UI animations unless you intentionally want a robotic, technical feel. Prof. Teeters taught me to avoid this for most interactions!

Ease: Default behavior (balanced)

Starts medium speed, accelerates middle, slows down end

This is the default if you don't specify a timing function. It's balanced but not optimized for any particular use case. Prof. Teeters taught me to always specify timing explicitly rather than relying on the default. For most UI interactions, ease-out is better than ease.

Ease-in: Slow start (feels hesitant)

Starts slow, gradually accelerates, ends at full speed

Feels like gathering momentum or winding up. Not ideal for hover states because the slow start makes it feel unresponsive. Prof. Teeters taught me to use this for elements leaving the screen ‚Äî the acceleration gives a sense of something being pulled away or dismissed.

Ease-out: Fast start (RECOMMENDED)

Starts fast, gradually decelerates, settles smoothly

This is Prof. Teeters' go-to recommendation for 95% of UI interactions. The fast start means users see immediate visual feedback when they hover or click. The slow end feels like natural settling into the final state. Mimics real-world physics where moving objects naturally slow down. Feels most responsive!

Ease-in-out: Slow both ends (long movements)

Starts slow, speeds up middle, ends slow (symmetrical)

Good for longer movements where an element travels across the screen. The symmetrical curve feels balanced for transitions that go from point A to point B. Less responsive feeling than ease-out because of the slow start. Prof. Teeters taught me to use this sparingly!

Which timing should you choose?

Ease-out for almost everything

Prof. Teeters' rule is simple: start with ease-out for all hover states, focus states, and appearing elements. Only use other timing functions when you have a specific reason. Linear for intentional robotic feel. Ease-in for dismissals. Ease-in-out for cross-screen movement. But ease-out is the default choice!

Why ease-out wins for UI: The psychology of responsiveness

Prof. Teeters explained that ease-out isn't just technically better ‚Äî it's psychologically better. Users need immediate visual feedback when they interact with your interface. The fast start of ease-out shows instant reaction to user input, while the slow ending feels like natural settling. This mimics real-world physics where things naturally decelerate, creating a satisfying interaction feel!

Why ease-out feels responsive:
/* Recommended pattern for most UI interactions */

  /* Slow end = smooth settling */
}

.button:hover {

Understanding cubic-bezier curves

Prof. Teeters explained that all timing functions are secretly cubic-bezier curves under the hood. ease, ease-out, linear ‚Äî they're all just shortcuts for specific cubic-bezier values. Understanding bezier curves unlocks unlimited timing possibilities!

Standard timing functions as cubic-bezier values:
/* All timing functions are cubic-bezier under the hood */

/* Linear */
.linear {
  transition: transform 300ms linear;
  /* Same as: cubic-bezier(0, 0, 1, 1) */
  /* Constant speed throughout */
}

/* Ease (default) */
.ease {
  transition: transform 300ms ease;
  /* Same as: cubic-bezier(0.25, 0.1, 0.25, 1) */
  /* Balanced curve */
}

/* Ease-in */
.ease-in {
  transition: transform 300ms ease-in;
  /* Same as: cubic-bezier(0.42, 0, 1, 1) */
  /* Slow start, fast end */
}

/* Ease-out (RECOMMENDED) */
.ease-out {
  transition: transform 300ms ease-out;
  /* Same as: cubic-bezier(0, 0, 0.58, 1) */
  /* Fast start, slow end */
}

/* Ease-in-out */
.ease-in-out {
  transition: transform 300ms ease-in-out;
  /* Same as: cubic-bezier(0.42, 0, 0.58, 1) */
  /* Slow both ends */
}

The four cubic-bezier values

Cubic-bezier takes four values: cubic-bezier(x1, y1, x2, y2). These values control two control points that define the curve shape. The X axis represents time (0 to 1), and the Y axis represents progress (0 to 1). Prof. Teeters taught me that understanding these values helps, but you don't need to memorize them ‚Äî use DevTools to experiment visually!

What the values mean

X axis is time, Y axis is progress

The format cubic-bezier(x1, y1, x2, y2) defines two control points. The first point is (x1, y1) and the second point is (x2, y2). X values must stay between 0 and 1 (representing time from start to finish). Y values can go outside the 0-1 range, which creates bouncy or anticipation effects! Prof. Teeters taught me to focus on the visual curve, not the numbers.

Creating custom curves

Drag handles to create any motion pattern

In the DevTools curve editor, you drag the two control point handles to shape the curve. Dragging up makes the animation faster at that point. Dragging down makes it slower. Dragging beyond the top creates overshoot (bouncy). Dragging below zero creates pull-back (anticipation). The curve you see is exactly how the animation will move!

Common custom cubic-bezier curves

While the five standard curves work for most cases, custom bezier curves let you create specific feelings. Prof. Teeters showed me three common patterns that add personality to animations:

Custom cubic-bezier curve examples:
/* Bouncy - overshoots then settles (playful) */
.button-bouncy {
  background: hsl(32, 76%, 63%);
  transform: scale(1);
  transition: transform 300ms cubic-bezier(0.68, -0.55, 0.265, 1.55);
  /* Y value above 1 creates overshoot */
  /* Perfect for playful, energetic brands */
}

.button-bouncy:hover {
  transform: scale(1.1);
  /* Bounces past 1.1, then settles back */
}

/* Anticipation - pulls back first (emphasis) */
.button-anticipation {
  background: hsl(210, 50%, 50%);
  transform: translateX(0);
  transition: transform 400ms cubic-bezier(0.6, -0.28, 0.735, 0.045);
  /* Negative Y value creates pull-back */
  /* Feels like winding up before moving */
}

.button-anticipation:hover {
  transform: translateX(20px);
  /* Pulls back slightly, then moves forward */
}

/* Smooth elegant - Material Design style (professional) */
.button-smooth {
  background: hsl(220, 13%, 18%);
  transform: scale(1);
  transition: transform 300ms cubic-bezier(0.4, 0, 0.2, 1);
  /* Gentle curve, refined feeling */
  /* Google's Material Design easing */
}

.button-smooth:hover {
  transform: scale(1.05);
  /* Subtle, sophisticated motion */
}

/* ALWAYS include reduced motion for all custom curves */
@media (prefers-reduced-motion: reduce) {
  .button-bouncy,
  .button-anticipation,
  .button-smooth {
    transition: none;
  }
}

Bouncy curves (playful)

Y values above 1 create overshoot

When the Y value goes above 1, the animation overshoots its target, then bounces back and settles. This creates a spring-like, energetic feeling. Perfect for playful brands, kids' products, or fun interactions. Prof. Teeters taught me to use this sparingly ‚Äî too much bounce feels cartoonish!

Anticipation curves (emphasis)

Negative Y values create pull-back

When the Y value goes below 0, the animation pulls back before moving forward ‚Äî like winding up. This mimics real-world physics where you pull back before throwing. Adds emphasis and character. Good for important actions where you want to draw attention!

Timing and brand personality

After I learned the technical aspects of timing functions, Prof. Teeters taught me something that changed my perspective completely: timing isn't just technical, it's emotional. The same button with different timing functions can feel playful, professional, friendly, or technical. Timing communicates brand values and personality just as much as colors and typography do!

Timing as emotional expression

Prof. Teeters explained that users don't consciously think about timing functions, but they feel them. Fast, bouncy timing feels energetic and playful. Slow, smooth timing feels professional and calm. The timing choices you make create emotional responses that reinforce or contradict your brand identity. This isn't just about making animations work ‚Äî it's about making them communicate!

Fast timing for energetic brands

Short duration (200-250ms), bouncy curves

When brands want to feel playful, youthful, or energetic, fast timing with bouncy cubic-bezier curves creates that personality. Overshooting animations, quick responses, snappy interactions ‚Äî all of these communicate energy and fun. Perfect for kids' products, gaming, entertainment, or casual applications. Prof. Teeters taught me that speed equals energy in users' minds!

Slow timing for calm brands

Longer duration (400-500ms), smooth curves

Professional services, luxury brands, and sophisticated products use slower timing with gentle ease-in-out curves. The unhurried pace feels refined and deliberate. Nothing rushes, nothing bounces ‚Äî every transition is smooth and controlled. Healthcare, finance, and high-end brands use this timing to communicate trustworthiness and sophistication!

Medium timing for friendly brands

Standard duration (300-350ms), ease-out

Most e-commerce, educational products, and general-purpose applications use medium timing with ease-out curves. It's balanced ‚Äî not aggressive, not sluggish. Approachable and versatile. This is the safe default when you want broad appeal without strong personality in either direction. Prof. Teeters called this the "Swiss army knife" of timing choices!

Linear timing for technical brands

Any duration, constant speed (precise)

Developer tools, industrial equipment, and data dashboards sometimes use linear timing intentionally. The robotic, mechanical feel communicates precision and technical accuracy. This is the one case where linear timing is a deliberate choice, not a mistake. When you want to emphasize logic over emotion, linear can work!

Why the playful button works

Fast + bouncy = energetic personality

The 250ms duration is snappy and immediate. The bouncy cubic-bezier curve with Y value above 1 creates an overshoot effect ‚Äî the button scales past 1.1, then bounces back and settles. This playful motion feels fun and energetic, perfect for kids' products or casual brands. Users subconsciously feel the energy!

Why the professional button works

Slow + smooth = sophisticated personality

The 400ms duration is deliberate and unhurried. The ease-in-out curve is symmetrical and smooth ‚Äî no sudden starts or stops. Adding opacity change creates visual sophistication. The subtle 1.05 scale (instead of 1.1) feels refined, not aggressive. Perfect for professional services or luxury brands!
