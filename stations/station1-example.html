Notice how the mobile styles are simple and clean? Then I ENHANCE for larger screens. This is called "mobile-first" and it's way less frustrating!

Understanding Media Queries

Prof. Teeters explained media queries like this: "Think of them as conditional styling - CSS that only applies when certain conditions are true."

Basic media query structure:
/* Base styles (mobile) - no media query needed */
.card {
  width: 100%;
  padding: 1rem;
}

/* Tablet and up */
@media (min-width: 768px) {
  .card {
    width: 50%;
  }
}

/* Desktop and up */
@media (min-width: 1024px) {
  .card {
    width: 33.333%;
  }
}

I learned that mobile styles don't need a media query - they're the default! Then you ADD styling for larger screens. This is the mobile-first approach.

Try It Yourself - Make a Responsive Card

Your Mission: Create a card that stacks vertically on mobile but displays side-by-side on desktop.

Starting HTML:
<div class="card">
  <img src="image.jpg" alt="Example image">
  <div class="card-content">
    <h3>Card Title</h3>
    <p>Card description text.</p>
  </div>
</div>

Your Challenge:

  1. Write mobile-first CSS (stacked layout)
  2. Add a media query for screens 768px and wider
  3. Use flexbox to make image and content side-by-side

‚úÖ HAP's Solution:

Mobile-first CSS:
/* Mobile: Everything stacks (default) */
.card {
  border: 1px solid hsl(0, 0%, 80%);
  border-radius: 8px;
  overflow: hidden;
}

.card img {
  width: 100%;
  display: block;
}

.card-content {
  padding: 1rem;
}

/* Desktop: Side-by-side with flexbox */
@media (min-width: 768px) {
  .card {
    display: flex;
  }

  .card img {
    width: 40%;
    object-fit: cover;
  }

  .card-content {
    width: 60%;
  }
}

How it works:

  • Mobile (default): Everything stacks naturally because block elements are 100% wide
  • Desktop (768px+): Flexbox makes items sit side-by-side, image takes 40%, content takes 60%
  • Why it's better: Mobile works without ANY media query! We only add code for desktop.

Quick Reference

1

Start Mobile-First

Write base styles for mobile, then ADD complexity with media queries for larger screens.

2

Common Breakpoints

768px (tablet), 1024px (desktop), 1440px (large desktop). But test YOUR content to find natural breaks!

3

Use Relative Units

rem, em, %, vw scale better than px.

4

Test Real Devices

DevTools is great, but nothing beats testing on an actual phone or tablet!

5

Flexible Images

max-width: 100% on images makes them scale down automatically.

6

Viewport Meta Tag

Always include: <meta name="viewport" content="width=device-width, initial-scale=1.0">
